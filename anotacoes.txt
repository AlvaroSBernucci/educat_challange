PIP:
Gerenciador de pacotes do python

VENV (Ambiente virtual):
É uma cópia da instalação do python onde cada VENV pode ter uma versão de ferramentas.
Tem dependência somente daquele projeto

Project e App:

App é um pedaço do projeto como um todo, que tem tudo aquilo que é relacionado a um contexto.
Ex: App de User | App de artigos, cada parte gerencia algo

Project (criar como setup) é o conjunto de várias apps que conversam entre si.


manage.py:
É o arquivo necessário para carregar todo o contexto de execunção do jungle

Apps:

É necessário colocar as apps criadas no project no arquivo settings.py
de preferencia com o caminho todo Ex:'Pasta do app -> apps -> nome da classe de configuração'

Criar migração:
python manage.py makemigrations

Rodar migração:
python manage.py migrate


Python decouple:
Biblioteca de python para esconder arquivos de configuração (Segurança)
Instalaçao:
  pip install python-decouple
  Criar arquivo .ENV na parte inicial do projeto
  Colocar os blocos de código desejados no arquivo removendo aspas e espaços
  no settings.py importar o decouple com: from decouple import config
  na parte de config, trocar o código pelo nome da variavel
  Ex: SECRET_KEY = config("SECRET_KEY")
  Adicionar o .env no gitignore
